## DOM
DOM(Document Object Model)은 HTML, XML 문서의 프로그래밍 interface라고 한다. 쉽게 이해하면, 웹 페이지를 이루는 요소들(태그 등)을 자바스크립트가 이용을 할 수 있게 브라우저 내에서 `트리 구조`로 만든 객체 모델이라고 생각하면 될 것 같다. 
- 즉, DOM은 HTML, XML 등의 문서와 JavaScript를 이어준다.

## VirtualDOM
실제 DOM(문서 객체 모델)의 가벼운 자바스크립트 객체 복사본으로, 실제 DOM 조작 전에 메모리상에서 변경 사항을 비교하여 최소한의 실제 DOM 업데이트만 수행함으로써 웹 애플리케이션의 렌더링 성능을 향상시키는 기술이다.

예를 들어 인스타그램, 페이스북, 유튜브 같은 서비스는 무한 스크롤로 데이터를 불러와 UI를 계속 업데이트한다. 하지만 실제 DOM을 직접 계속 수정하면 리페인트가 자주 발생해 성능 저하와 서비스 이슈로 이어질 수 있다.

- 실제 DOM에 직접 접근하는 대신에, 이를 자바스크립트 객체로 구성하여 DOM을 추상화하여 사용하는 방식이다.
- React를 사용할 때에는 React가 모두 처리를 해주기 때문에, DOM API를 직접 구현하지 않아도 된다.

### 장점
- **성능 향상** : 실제 DOM 조작의 빈도를 줄이고 렌더링 과정을 최적화하여 애플리케이션의 성능을 높인다.
- **개발 효율성** : 개발자는 DOM 조작과 관련된 복잡한 작업을 직접 할 필요 없이, 상태 변화에 집중하여 UI를 개발할 수 있다.
- **비용 절감** :  브라우저가 불필요한 렌더링 작업을 줄여 CPU 사용량과 연산 비용을 절감할 수 있다.
