# 서브쿼리란
**한 쿼리 내에 포함된 또 하나의 쿼리**
![image](/image/서브쿼리.png)

- 서브쿼리는 메인쿼리에 종속해있으며 메인쿼리 없이는 단독적으로 실행될 수 없다.
-  서브쿼리는 메인쿼리의 칼럼을 사용할 수 있지만, 반대로 `메인쿼리는 서브쿼리의 칼럼을 사용할 수 없다.` 
- 서브쿼리의 종류는 크게 3가지 (스칼라, 인라인 뷰, 중첩)로 어느 위치에서 사용하느냐에 따라 분류된다.

| 종류  | 설명 |
| --- | --- |
| **스칼라 서브쿼리**(Scalar Suquery) | **SELECT** 절에 위치하며 한 레코드당 정확히 **하나의 값을 반환**하는 서브쿼리 (단일행, 단일칼럼을 반환) |
| **인라인 뷰**(Inline View) | **FROM** 절에 위치하는 서브쿼리로 결과는 반드시 하나의 테이블로 리턴되어야 한다. |
| **중첩 서브쿼리**(Nested Subquery) | **WHERE** 절에 위치하며 **결과집합을 한정**하기 위한 서브쿼리로, 서브쿼리가 메인쿼리 컬럼을 참조할때 상관관계 있는 서브쿼리라고 한다. (단일행, 다중행을 반환) |

## 스칼라 서브쿼리 (Scalar Subquery)
- SELECT 절에 위치하며 칼럼의 역할을 하므로 해당 서브쿼리의 결과는 반드시 단일 행 or 단일 값으로 리턴되어야 한다.
- 다중 행 값이 조회되면 `"ORA-01427: single-row subquery returns more than one row"` 라는 오류가 발생하므로 주의
```sql
SELECT name, (
    SELECT score.math
    FROM score
    WHERE score.id = student.id
    ) AS score
FROM student 
WHERE name = 'KIM';
```

## 인라인 뷰 (Inline View)
- FROM 절에 위치하며 테이블의 일부 데이터만을 불러와 복잡한 쿼리를 단순화 할수 있으며, 쿼리의 가독성이 높다.
- 해당 서브쿼리의 결과는 반드시 하나의 테이블로 리턴되어야 한다.
- 인라인 뷰는 SQL문이 실행될때 임시적으로 생성되는 동적인 뷰 로 이와 같은 뷰를 Dynamic View 라고 부른다.
- **주의점**: SQL을 좀더 빠른 속도로 사용하기 위한것이 인라인 뷰이지만, SQL을 구성할때 잘못 사용하게 되면 액세스 속도를 저하 시키는 경우가 많으므로 사용에 주의
```sql
SELECT e.name, e.title, d.name
FROM (SELECT name, title, dept_id
      FROM s_emp
      WHERE title = '사원') e, s_dept d
WHERE e.dept_id = d.id;
```

## 중첩 서브쿼리 (Nested Subquery)
- WHERE 절에서 사용하는 서브쿼리로 메인쿼리 테이블의 특정 컬럼 값과 비교한 값을 반환하는 용도로 사용된다.
- 단일행 / 다중행 둘 다 리턴이 가능하다.

### Single Row SubQuery
- Single Row SubQuery란 서브쿼리에서 메인쿼리로 전달되는 행이 단 하나인 경우를 말한다.
- WHERE 절에 단일 행 연산자를 사용한다.
```sql
SELECT name, title, dept_id
FROM s_emp
WHERE dept_id = (SELECT dept_id
                 FROM s_emp
                 WHERE name = 'jk');
```

### Multi Row SubQuery
- Multi Row SubQuery란 서브쿼리에서 메인쿼리로 전달되는 행이 여러 개인 경우를 말한다.
- WHERE 절에 다중 행 연산자를 사용한다.
```sql
SELECT name, dept_id
FROM s_emp
WHERE dept_id IN (SELECT id
                  FROM s_dept
                  WHERE region_id = 3);
```

### Multi-Column SubQuery
- Single Row SubQuery란 서브쿼리 구문을 작성할 때 WHERE 절에서 비교하는 칼럼이 하나가 아니라 여러 개의 칼럼을 동시에 비교하는 경우를 말하며 이런 경우를 pair-wise 되었다고 한다.
```sql
SELECT name, dept_id, salary
FROM s_emp
WHERE (dept_id, salary) IN (SELECT dept_id, MIN(salary)
                            FROM s_emp
                            GROUP BY dept_id);
```

## 기타 서브쿼리

### HAVING 절에서의 SubQuery
```sql
SELECT dept_id, AVG(salary)
FROM s_emp
GROUP BY dept_id
HAVING AVG(salary) > (SELECT AVG(salary)
                      FROM s_emp
                      WHERE dept_id = 100);
```

### CREATE 절에서의 SubQuery
- 테이블을 생성할 때 이미 만들어져 있는 기존의 테이블을 이용해서 특정 컬럼 또는 행만으로 원하는 테이블을 생성할 수 있다.
- 데이터는 제외하고 테이블만 생성하려면 조건절이 항상 거짓이게 하면 된다. → ex. WHERE 1 <> 1
```sql
CREATE TABLE emp_100 (id, name, mailid, start_date)
AS SELECT id, name, mailid, start_date
FROM s_emp
WHERE dept_id = 100;

Table created.
```

### DML 문에서의 SubQuery
```sql
INSERT INTO emp_100
	(id, name, mailid, start_date)
SELECT id, name, mailid, start_date
FROM s_emp
WHERE start_date < '24/05/18';
```

```sql
UPDATE s_emp
SET dept_id = (
    SELECT dept_id
    FROM s_emp
    WHERE title = '과장')
WHERE name = 'jk';
```
